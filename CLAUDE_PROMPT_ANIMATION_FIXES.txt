# EXECUTION BRIEF: Animation Architecture & Bug Fixes

@claude Execute the following implementation brief exactly as written. Do not deviate. Do not redesign the visual brand.

## 1. OBJECTIVE
- Audit and fix broken or fragile animation behavior in the Shopify theme.
- Standardize animation ownership so there is one reliable implementation path for hero reveal, stat counters, scroll reveals, and ticker behavior.
- Remove fake or duplicate animation logic.

## 2. CURRENT PROBLEM
- **Hero animation contract is broken:**
  - `assets/pulsar-animations.js` expects `window.pulsarRevealHero()` to exist.
  - `sections/pulsar-hero.liquid` does not actually define that function.
  - Result: intended premium hero reveal is not implemented; fallback fade is doing the work.
- **Stat counters are implemented in multiple places:**
  - `assets/pulsar-enhancements.js` contains global counter logic.
  - `sections/pulsar-stats-counter.liquid` also contains inline counter logic.
  - Result: duplicate ownership, race-condition risk, inconsistent behavior.
- **Stats selector mismatch:**
  - `assets/pulsar-animations.js` targets `#pulsar-stats`.
  - Actual section id is `#pulsar-stats-{{ section.id }}`.
  - Result: dedicated stats section animation block never matches the real section.
- **Member ribbon ticker lacks lifecycle cleanup:**
  - `sections/pulsar-member-ribbon.liquid` starts a perpetual `requestAnimationFrame` loop and adds listeners.
  - No teardown logic for Shopify editor section unload/re-render.
  - Result: possible duplicate loops/listeners in theme editor.
- **Marquee depends on external JS:**
  - Uses `marquee.js` and `var(--marquee-speed)`, but speed is not defined in the section itself.
  - Custom CSS in Shopify is insufficient for these behavior bugs because these problems require JavaScript, section scoping, and Shopify lifecycle handling.

## 3. IMPLEMENTATION STRATEGY
- Make `assets/pulsar-animations.js` the single source of truth for motion behavior.
- Keep `assets/pulsar-enhancements.js` limited to non-animation enhancements only.
- Remove inline JS behavior from sections where a global controller is more reliable.
- Implement real hero animation behavior instead of fallback pretending (either define `window.pulsarRevealHero()` or move logic entirely to global scope).
- Fix stats targeting by using section-scoped selectors.
- Add proper Shopify editor lifecycle safety for ticker animations (`shopify:section:unload`).
