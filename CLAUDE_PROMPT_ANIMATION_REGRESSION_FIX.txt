## 1. OBJECTIVE

## 2. CURRENT PROBLEM
- **Hidden Stats Numbers:** The stats numbers are still present in the Liquid markup, but users report they are gone on the homepage.
- **GSAP Pattern Mismatch:** `sections/pulsar-stats-counter.liquid` hides `.pulsar-reveal` elements by default (`opacity: 0`, `translateY(...)`). `assets/pulsar-animations.js` uses `gsap.from(...)` for generic `.pulsar-reveal` elements. This is fragile because `gsap.from(...)` is the wrong pattern for elements whose final state is not already visible in CSS.
- **Duplicate Ownership:** `assets/pulsar-animations.js` separately animates `.pulsar-stats__item`. The items are controlled twice (by generic `.pulsar-reveal` logic AND dedicated stats-section reveal logic).
- **Split Fallback:** `assets/pulsar-enhancements.js` still contains fallback reveal logic for `.pulsar-reveal`, leaving ownership partially split.

## 3. IMPLEMENTATION STRATEGY
- **Unambiguous Ownership:** Do not allow stats items to be animated by both generic and stats-specific reveal systems. Generic `.pulsar-reveal` handles visual reveal. Stat counters handle number animation. Remove special stats-item reveal animation unless absolutely necessary.
- **Replace Fragile GSAP:** Replace fragile `gsap.from(...)` patterns on hidden-by-default elements with `gsap.fromTo(...)` or `gsap.to(...)` using an explicit final visible state.
- **Independent Counters:** Keep counters independent and only animate once.
- **Preserve Fallbacks:** Preserve reduced-motion fallback.
