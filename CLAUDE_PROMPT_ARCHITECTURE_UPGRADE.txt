# EXECUTION BRIEF: Architecture Upgrade & Lifecycle Reliability

@claude Execute the following implementation brief exactly as written. Do not deviate.

---

## 1. OBJECTIVE
- Improve the Shopify theme architecture beyond the initial animation bug fixes.
- Focus on maintainability, Shopify editor reliability, configurability, and runtime dependency discipline.
- Reduce hardcoded content and reduce cross-file ownership confusion.

## 2. CURRENT PROBLEM
- **Global animation behavior is still not fully Shopify-section-lifecycle aware:**
  - `sections/pulsar-member-ribbon.liquid` handles section lifecycle events.
  - `assets/pulsar-animations.js` and `assets/pulsar-enhancements.js` do not show equivalent global re-init/destroy handling for section reloads.
  - Result: theme editor can become inconsistent after section add/remove/reload.
- **Reveal ownership is still split:**
  - `assets/pulsar-animations.js` owns GSAP reveal behavior.
  - `assets/pulsar-enhancements.js` still contains fallback reveal logic for `.pulsar-reveal`.
  - Result: architecture is improved but still not cleanly single-owner.
- **Lenis is loaded and initialized globally on desktop and tied into GSAP ticker:**
  - This is a real behavioral dependency, not a minor enhancement.
  - Result: more moving parts, more failure/debug surface, unclear ROI unless intentional.
- **Content is still hardcoded in sections:**
  - `sections/pulsar-member-ribbon.liquid` hardcodes the member list in Liquid.
  - `sections/pulsar-stats-counter.liquid` hardcodes stat labels/values.
  - Result: poor maintainability and weak Shopify editor usability.
- **Theme loads third-party sources heavily:**
  - `theme.liquid` loads GSAP, ScrollTrigger, Lenis, and Google Fonts from third-party sources.
  - Result: unnecessary external runtime dependency risk, harder debugging.
- **Ticker behavior is partially hardcoded:**
  - member ribbon speed is hardcoded in JS.
  - Result: poor configurability.

## 3. IMPLEMENTATION STRATEGY
- **Introduce an editor-safe section lifecycle architecture:**
  - Create a global init/destroy pattern for Pulsar sections.
  - Support Shopify editor `shopify:section:load/unload/reorder` events.
- **Consolidate reveal ownership:**
  - One file must own all reveal logic, including fallback behavior. The other must become utility-only.
- **Make content Shopify-native and editor-friendly:**
  - Stats must use blocks/settings.
  - Member ribbon must use blocks or a configurable content source. Parameterize motion settings (e.g., ticker speed).
- **Decide Lenis strategy explicitly:**
  - Either keep it as a supported dependency with proper lifecycle handling or remove it if not absolutely essential for brand premium feel.
